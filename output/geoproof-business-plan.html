Executive summary

What it is

GeoProof is a verifiable crop-condition network designed for Southeast Asian agricultural insurance. It combines simulated field sensors, optional drone or satellite-derived observations, a Solana-based proof registry, and USDC settlement for enterprise data access.

The initial product is not a full remote-sensing marketplace. It is a tight vertical workflow focused on one job: make crop-damage validation faster, cheaper, and more trustable.

Why it wins

  • Narrow wedge: insurance verification instead of generic Earth-data marketplace.
  • Hackable MVP: simulation is enough; no dependency on live satellites for demo value.
  • Compelling token story: reward nodes for useful evidence, not speculative noise.
  • Climate framing: resilience, transparency, and public-good environmental infrastructure.
Target users
Agricultural insurers, reinsurers, agribusiness lenders, cooperatives, climate-risk programs
Initial revenue
USDC per verification request, API access, partner pilots, annual enterprise contracts
MVP timebox
6–7 days, roughly 30–36 focused solo hours, vibe-coded but still demo-safe

Problem and opportunity

Core problem

Agricultural insurance claims are slow and disputed because evidence is fragmented. Farmers, insurers, field adjusters, and data vendors often rely on inconsistent timestamps, unverifiable media, and delayed site visits.

When claims are triggered by flood, drought, disease, or heat stress, the highest-value evidence exists in short time windows. Missing or manipulated records increase loss ratios and reduce trust.

Market opening

Southeast Asia has expanding climate-risk exposure, fragmented agricultural data infrastructure, and many crop systems where on-the-ground proof still matters. That makes this region a strong starting point for a proof network that mixes remote sensing and field observations.

GeoProof enters as infrastructure, not a full insurer. That keeps scope light and makes B2B sales clearer.

Positioning statement

GeoProof is the verifiable evidence layer for crop-insurance decisions: fast enough for operations, transparent enough for auditors, and simple enough to launch as a solo-built MVP on Solana devnet.

Business model

Revenue streams

  1. Verification fee: insurer pays per claim validation request in USDC.
  2. Enterprise subscription: recurring access to dashboard, APIs, audit trail, and analytics.
  3. Data partnership fee: custom feeds for co-ops, lenders, or climate programs.
  4. White-label integration: workflow or widget integration into insurer systems.

Cost structure

  • Solana transaction and storage costs.
  • RPC and indexing service costs.
  • Cloud hosting for dashboard and worker jobs.
  • Remote-sensing data and drone partnerships later.
  • Legal/compliance overhead when moving beyond prototype.

Product requirements document

Product goal

Enable enterprise users to request verifiable crop-condition evidence for a location and date range, pay on-chain in USDC, and receive a transparent evidence trail plus confidence score, while rewarding contributing data nodes.

Primary jobs to be done

  • As an insurer, I want to verify whether a farm likely experienced crop stress during a claim window.
  • As a node operator, I want to earn rewards for submitting useful, timely, location-linked evidence.
  • As an admin, I want to review network quality and prevent spam data.
  • As an auditor, I want a tamper-evident record of when and where evidence was submitted.

Success criteria

  • Claim verification request can be created in under 2 minutes.
  • At least one full end-to-end demo flow works on Solana devnet.
  • Dashboard clearly visualizes node activity and evidence timeline.
  • Reward distribution completes automatically after verification event.

Functional requirements

Non-functional requirements

  • Clear wallet onboarding for non-crypto demo audience.
  • Devnet reliability with graceful fallback when RPC is slow.
  • Readable UI on desktop and tablet during judging/demo.
  • Simple, explainable scoring logic instead of opaque ML.
  • All important actions visible in activity history.

Out of scope

  • Real legal insurance adjudication.
  • Production tokenomics with open market liquidity.
  • Consumer sign-up flows.
  • Large-scale node slashing or advanced oracle consensus.

Technical architecture

System overview

GeoProof uses a hybrid architecture: lightweight on-chain records for integrity and payment, plus off-chain storage and services for evidence payloads, maps, and scoring. This keeps the MVP cheap, explainable, and fast to ship.

On-chain

  • Solana devnet
  • Anchor program
  • SPL reward token mint
  • USDC transfer flow
  • Accounts for nodes, requests, rewards

Off-chain

  • Next.js dashboard
  • Python or TS simulator workers
  • JSON evidence store
  • Optional IPFS/Pinata later
  • Map rendering and request indexing

Data logic

  • Geotag + timestamp validation
  • Evidence scoring
  • Node reputation stub
  • Reward allocation logic
  • Report generation

Reference stack

Layer Choice Reason
Blockchain Solana + Anchor Fast devnet demo, low fees, good wallet ecosystem
Frontend Next.js + Tailwind + wallet adapter Fast UI iteration and clean dashboard delivery
Backend jobs Python scripts or Node workers Easy simulation and event scheduling
Storage Local JSON for demo, IPFS-ready abstraction Move fast without external dependency risk
Maps Mapbox or Leaflet Node visualization and claim regions
Indexing Simple RPC polling or lightweight parser Enough for MVP visibility

Anchor account design

  • NodeAccount: wallet, node type, region, metadata URI, status.
  • EvidenceAccount: node, request ref, hash, time, location, metrics summary.
  • VerificationRequest: insurer wallet, farm ID, claim window, fee, state.
  • RewardVault: token mint, pool balance, distribution state.

Security choices for MVP

  • Keep payment flows simple and scoped.
  • Use admin authority for pause/allowlist at MVP stage.
  • No unaudited staking or leveraged token logic.
  • Record hashes on-chain; keep large payloads off-chain.

Integrations

MVP integrations

  • Phantom / Solflare wallet connection
  • Solana wallet adapter
  • Devnet USDC test flow
  • Map provider for farm and node locations
  • Basic RPC endpoint

Phase 2 integrations

  • Pinata/IPFS for evidence payload pinning
  • Webhook or email notification service
  • Commercial weather or remote-sensing APIs
  • Claims system export for enterprise partner
  • Drone upload portal

Integration policy

Every external integration should sit behind an adapter layer. For the hackathon build, simulate upstream satellite and field data but structure code so real providers can replace the simulator without rewriting the contract or dashboard.

Operations, risks, and KPIs

Core KPIs

  • Number of active nodes
  • Evidence submissions per day
  • Verification requests completed
  • Median claim response time
  • Evidence confidence score coverage
  • USDC revenue per customer

Main risks

  • Fake or low-quality node data
  • Overcomplicated token model
  • Regulatory confusion around insurance promises
  • Dependence on external data vendors too early
  • Judges misunderstanding why blockchain is needed

Mitigations

  • Use allowlist and admin moderation in MVP.
  • Frame token as network reward, not investment.
  • State clearly that GeoProof supports decisions, not underwriting itself.
  • Simulate first, integrate later.
  • Show immutable audit trail + payment automation as blockchain benefit.

Demo and investor framing

3-minute demo script

  1. Open dashboard and show map with registered nodes.
  2. Create a verification request for a farm region and date window.
  3. Run simulator so evidence appears on-chain and in the event feed.
  4. Show confidence score and evidence record linked to wallet addresses.
  5. Trigger USDC payment and reward distribution to contributing nodes.

Why Solana belongs here

  • Low-fee, high-speed settlement for many small evidence events.
  • Transparent record of who submitted what and when.
  • Programmable incentives for distributed data providers.
  • Stablecoin payments for enterprise-friendly business flow.

Final founder checklist

  • One clear vertical: agri-insurance verification.
  • One working flow: request → evidence → score → payment → reward.
  • One clear message: climate proof infrastructure, not generic geo-data marketplace.
  • One backup path: pre-seeded data and recorded demo in case devnet misbehaves.
